# Configuration

`qoder-otel-plugin` reads its runtime configuration from:

```text
<QODER_HOME>/gtrace.json
```

Minimal configuration:

```json
{
  "enabled": true,
  "endpoint": "http://localhost:4318",
  "tracePath": "v1/traces",
  "metricsPath": "v1/metrics"
}
```

Common fields:

| Field | Description |
| --- | --- |
| `enabled` | Enables upload when `true`. |
| `endpoint` | Receiver base URL. |
| `tracePath` | Trace route appended to `endpoint`. |
| `metricsPath` | Metrics route appended to `endpoint`. |
| `otel_traces_url` | Full trace URL. Overrides `endpoint + tracePath` when set. |
| `otel_metrics_url` | Full metrics URL. Overrides `endpoint + metricsPath` when set. |
| `headers` | Extra HTTP headers, for example `X-Token`. `To-Headless: "true"` is added by default. |
| `public_key` / `secret_key` | Optional Basic auth credentials. |
| `resourceAttributes` | Extra OTLP resource attributes. |
| `max_chars` | Maximum size for captured input/output fields. Default `20000`. |
| `timeout_ms` | HTTP upload timeout. Default `25000`. |
| `debug` | Enables extra hook logs when supported. |
| `fail_on_error` | Re-throws hook upload errors when `true`. |

Local state and logs:

| Path | Purpose |
| --- | --- |
| `<QODER_HOME>/qoder-otel-hook.log` | Upload and parsing diagnostics. |
| `<QODER_HOME>/qoder-otel-native-hook-events.jsonl` | Raw Qoder hook events captured by the plugin. |
| `<QODER_HOME>/qoder-otel-state/` | Duplicate-upload guards. |

Path layout by variant:

| Variant | Qoder home | Config root |
| --- | --- | --- |
| `cn` | `~/.qoder-cn` | `~/.config/QoderCN` |
| `global` | `~/.qoder` | `~/.config/Qoder` |

Runtime files are stored under `<QODER_HOME>`:

| File | Example |
| --- | --- |
| `gtrace.json` | `~/.qoder-cn/gtrace.json` |
| `qoder-otel-hook.log` | `~/.qoder-cn/qoder-otel-hook.log` |
| `qoder-otel-native-hook-events.jsonl` | `~/.qoder-cn/qoder-otel-native-hook-events.jsonl` |
| `qoder-otel-state/` | `~/.qoder-cn/qoder-otel-state/` |

Important upload log messages in `qoder-otel-hook.log`:

| Message | Meaning |
| --- | --- |
| `prepared qoder spans` | Transcript, token rows, hook events, and span counts before upload. |
| `upload traces start` | Target URL, body size, timeout, and redacted headers before HTTP upload. |
| `upload traces success` | HTTP status, duration, and short response body after successful upload. |
| `upload traces http_error` | Non-2xx HTTP response with status and short response body. |
| `upload traces exception` | Network, timeout, or encoding exception. |
| `upload metrics start` | Target URL, metric point count, metric names, body size, timeout, and redacted headers before HTTP upload. |
| `upload metrics success` | HTTP status, duration, and short response body after successful metrics upload. |
| `upload metrics http_error` | Non-2xx metrics HTTP response with status and short response body. |
| `upload metrics exception` | Metrics network, timeout, or encoding exception. |
| `uploaded qoder spans` | Final trace id, span names, operation names, and upload response. |
| `skipped duplicate upload` | Same transcript fingerprint was already uploaded. |

Sensitive headers such as `Authorization` and `X-Token` are redacted in logs.

Default paths follow native OTLP HTTP conventions:

| Signal | Default path |
| --- | --- |
| traces | `v1/traces` |
| metrics | `v1/metrics` |

GTrace/Dataway paths such as `v1/write/otel-llm` and `v1/write/otel-metrics` should be configured explicitly in `gtrace.json` or generated by the installer when using the GTrace preset.

Installer presets:

| Type | Trace path | Metrics path |
| --- | --- | --- |
| `otlp` | `v1/traces` | `v1/metrics` |
| `gtrace` | `v1/write/otel-llm` | `v1/write/otel-metrics` |

Example:

```bash
bash scripts/install-local.sh --type gtrace --variant cn --endpoint https://llm-openway.guance.com --x-token <token>
```

Data identity:

| Attribute | Source |
| --- | --- |
| `gen_ai.conversation.id` / `session_id` | Qoder `session_id`, aligned with `chat_message.session_id`. |
| `gen_ai.turn.id` / `turn_id` | Qoder `request_id`, treated as turn id. |
| `qoder.request_id` | Compatibility copy of the same turn id. |

LLM token fields:

| Attribute | Meaning |
| --- | --- |
| `gen_ai.usage.input_tokens` | Per-call input tokens reported by Qoder `prompt_tokens`. This is the real input size for that LLM request. |
| `gen_ai.usage.cache_read.input_tokens` | Per-call cached input tokens reported by Qoder. This is a subset of `prompt_tokens`, not a value to subtract from input tokens. |
| `gen_ai.usage.output_tokens` | Per-call completion tokens. |
| `qoder.usage.prompt_tokens` | Raw Qoder `prompt_tokens`, the full prompt/context size for that LLM request. |

Metrics:

| Metric | Type | Unit | Source |
| --- | --- | --- | --- |
| `gen_ai.workflow.duration` | Histogram | `s` | `invoke_agent` span duration |
| `gen_ai.agent.operation.count` | Sum | `-` | `llm`, `tool:*`, `skill:*` spans |
| `gen_ai.agent.operation.duration` | Histogram | `ms` | `llm`, `tool:*`, `skill:*` span duration |
| `gen_ai.client.token.usage` | Histogram | `{token}` | `llm` span input/output token attributes |

Metrics are derived from the same spans uploaded as traces. `assistant` spans do not generate metrics.
